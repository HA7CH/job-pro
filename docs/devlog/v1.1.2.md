## 修复内容

### pingan：`positionType` server-filter 上游静默失效

**文件：** `cli/src/pingan.ts`（+15/-3 行）

**根因：** 平安 API 的 `positionType` 请求参数上游悄悄失效——传任何值都返回 `data: null`，消息为"请求成功"（误导性）。v1.0.93 我们已经去掉了 keyword，但 scope 分类还依赖这个参数。

**修复：** 完全移除 `positionType` 请求参数，改为在客户端对返回数据按 `hireType` 字段过滤。

### mihoyo：campus channel ID 不存在

**文件：** `cli/src/mihoyo.ts`（+2/-1 行）

**根因：** scope=campus 映射到 `channelDetailIds: [2]`，但米哈游 API 的 channel `2` 不存在。实际上 campus 和 social 都走同一个 channel（id `1`），通过 `hireType`（0=社招, 1=校招）区分。

**修复：** 两种 scope 都使用 channel `[1]`，用 `hireType` 区分。

### oppo：campus 招聘类型字符串错误

**文件：** `cli/src/oppo.ts`（+7/-2 行）

**根因：** scope=campus 发送 `recruitmentType: "Campus"`，但 OPPO API 实际使用 `"Graduate"`（应届生）。

**修复：** 改为 `"Graduate"`。注：OPPO 目前应届生岗位极少（3 个），实习生岗位多（103 个），这是正常现象，不是 bug。

### zerooneai：无校招但未声明

**文件：** `cli/src/zerooneai.ts`（+15/-5 行）

**根因：** 零一万物（01.AI）的 Feishu 租户只声明了 `recruitment_type id: "1"`（社招/Experienced），根本没有校招渠道。`supportedScopes` 应去掉 `"campus"`。

**修复：** `supportedScopes` 改为 `["social", "all"]`，dispatcher 对 `--scope campus` 直接 fail-fast。
