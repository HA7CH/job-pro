## 修复内容

### tencent / jd / unitree：声明 `supportedScopes`

**文件：** `cli/src/tencent.ts`、`cli/src/jd.ts`、`cli/src/unitree.ts`

这三个 adapter 在 v1.1.0 未声明 `supportedScopes`，导致 dispatcher 无法 fail-fast，会对不支持的 scope 静默返回 campus 数据。补充声明后行为正确。

### liepin factory：声明 `supportedScopes: ["social", "all"]`

**文件：** `cli/src/liepin.ts`

Liepin 的 4 个 adapter（hikvision/cicc/cainiao/webank）都走猎聘渠道，只有社招，无校招/实习。

### jd 搜索结果为空（核心 bug）

**文件：** `cli/src/jd.ts`（+94 行）

**根因：** `searchPositions` 默认 `recruitType: "present"`，但 JD 的 `present`（应届）bucket 对大多数关键词返回 `totalNumber > 0, items: []`——数量不为零但列表为空，这是 JD API 的历史遗留 bug。

**修复：** `searchPositions` 现在并行 fan-out 三个 bucket（`present` + `internship` + `talent`），按 `publishId` 去重合并。scope 映射：
- `--scope intern` → 只查 `internship`
- `--scope campus` → 查 `present` + `talent`（应届，不含实习）
- `--scope social` → 查 `talent`（有经验社招）

### didi 校园/社招过滤丢失所有岗位

**文件：** `cli/src/didi.ts`（+5/-2 行）

**根因：** 过滤逻辑用 `"JR-2026"` 前缀匹配，但 didi 实际返回的是 `"JR2026"`（无连字符）。正则改为 `/^JR\d{4}/`。

### weibo 默认 channel 错误

**文件：** `cli/src/weibo.ts`（+9/-2 行）

**根因：** v1.1.0 把 weibo 的默认 channel 设为 `campus`，但 weibo 的 campus siteId `43534` 已下线（dark），导致裸命令 `weibo search` 返回空。

**修复：** 默认改为 `social`，`supportedScopes` 改为 `["social", "all"]`。
