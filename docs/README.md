# docs 目录说明

每次更新项目后，在对应文件夹里新建一个以版本号命名的 md 文件。**不要覆盖旧文件。**

---

## 📁 文件夹结构

```
docs/
├── changelog/        # 给用户看的更新说明（非技术语言）
│   └── v1.0.93.md
├── devlog/           # 给开发者看的技术日志
│   └── v1.0.93.md
└── README.md         # 本文件
```

---

## 📝 changelog/（用户更新说明）

**面向：** 使用 job-pro 的普通用户  
**语言风格：** 口语化，不写代码，不写文件路径  
**内容：** 修复了哪些 bug（用户感知层面）、新增了哪些功能、当前各公司支持状态  
**命名：** `v{版本号}.md`，例如 `v1.0.94.md`

---

## 🔧 devlog/（开发者技术日志）

**面向：** 接手项目的开发者（或未来的自己）  
**语言风格：** 技术语言，含文件路径、函数名、根因分析  
**内容：** 本次修复的 bug（含根因和涉及文件）、历史修复回顾、已知未解决问题、功能路线图、快速验证命令  
**命名：** `v{版本号}.md`，例如 `v1.0.94.md`

---

## ✏️ 每次更新后的操作清单

1. 确认当前版本号（`cli/package.json` → `"version"`）
2. 在 `docs/changelog/` 新建 `v{版本号}.md`，写用户更新说明
3. 在 `docs/devlog/` 新建 `v{版本号}.md`，写技术开发日志
4. 两个文件都不要覆盖旧版本的文件
