# v1.0.93 更新说明

> 发布日期：2026-05-19

---

## 🎉 这次更新解决了什么？

**一句话总结：简历匹配功能从「几乎全部返回 0 结果」修复为「46/49 家公司正常工作」。**

---

## 🐛 修复的问题

### 简历匹配返回 0 个岗位

**影响范围：** 字节跳动、阿里巴巴、网易、小米、滴滴、B 站、智谱 AI、京东、理想汽车等大量公司

**之前的表现：**  
上传简历后，`match` 命令对大多数公司返回空结果，即使你的技能（PyTorch、LLM、RAG 等）与这些公司的岗位完全匹配，依然一无所获。

**现在的表现：**  
系统会聪明地拆分简历关键词，分别搜索后合并结果，再统一评分排序。测试结果从几乎全部 0 条提升到每家公司返回 1–10 条相关岗位。

---

### 比亚迪、微博、vivo、拼多多返回无关岗位

**之前的表现：**  
这几家公司不支持关键词搜索，只能拉取全量岗位后本地匹配。当简历是英文而岗位标题是中文时，匹配分数全为 0，最终返回空列表。

**现在的表现：**  
当精确匹配为空时，自动兜底返回当前开放的岗位列表，不再一片空白。

---

### 京东 `match` 完全失败

**之前的表现：** 报错"no positions returned"，无论如何都拿不到结果。

**现在的表现：** 正常返回岗位。（京东校招 API 有特殊限制，已针对性修复。）

---

### 平安 `match` 报错

**之前的表现：** 使用英文关键词时 API 直接报错，导致命令失败。

**现在的表现：** 自动切换为全量拉取模式，绕过平安 API 对英文关键词的限制。

---

### 理想汽车 `match` 崩溃

**之前的表现：** 命令直接抛出 `Error: keyword is not defined` 错误，无法使用。

**现在的表现：** 已修复，正常返回匹配结果。

